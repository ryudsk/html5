  2. 추가기능

    1. 멀티미디어 - 별도 plug-in 없이 음악, 동영상 재생
    2. 그래픽 - 하드웨어 가속 2, 3차원 그래픽 구현(DOM API) ex) 앵그리버드
    3. 통신 - 단방향에서 양방향 통신으로
    4. 장치접근 - device 정보 수집 및 control가능
    5. 오프라인 및 저장소 - offline에서도 동작 App.
    6. 시멘틱(Semantic) 웹
      • 지능형 Web, 검색 엔진과 같은 프로그램이 정보의 의미를 분석하고 처리하여 제공
      • h는 제목, p는 문단 - 기능에 의미가 추가됨
      • 과거 layout을 div(또는 table)로 하던 것을 header, section, article, footer, nav 등 구조태그를 사용
    7. css완벽지원
    8. 개발환경
      • 모바일, 태블릿, 스마트TV
      • App Developer - 안드로이드(java), iOS(Objective C)
      • Publisher - HTML5

  3. 문법

    1. 속성과 속성은 공백으로 구분
    2. 속상에 값을 생략 가능(논리 속성)
    3. 대소문자 구분 없음
    4. ', " 혼용 가능
    5. 태그 중첩 불가 <p><strong><⁄p><⁄strong>
    6. 동일속성 중복선언 불가
    7. Entity code or name
    8. 권장 속성
      • alt, disabled - 폼을 사용하지 못하도록
      • href, id, src, style, title - href도움말
    9. 지원하지 않는 속성
      • align, bgcolor, border, color, hspace, vspace

  4. Contents model

    1. meta - 문서에 관한 정보나 다른 문서와의 관계, link, style, title, meta, script
    2. flow - meta를 제외한 대부분
    3. sectioning - 컨텐츠 구역 요소(제목, 푸터의 범위 지정), 장, 절, 칼럼, 섹션의 명시적 표현, section, article, aside, nav, (body(sectioning root))
    4. heading - 제목, h1~6, hgroup
    5. phrasing - 단락, h1~6, p
    6. embed - 외부 리소스, img, audio, video, canvas, embed, object, iframe
    7. interactive - 사용자 조작, a, audio, video, button, input, select, textarea, img, menu
